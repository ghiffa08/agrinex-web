1. Push ke repository:
   git push origin main

2. Deploy ke production server:
   ssh user@smartdrip-system.agrinex.io
   cd /var/www/agrinex-smartdrip
   git pull origin main
   npm run build
   php artisan config:clear
   php artisan cache:clear
   sudo systemctl restart php8.3-fpm
   sudo systemctl reload nginx

3. Test API endpoints (harusnya return 200 OK):
   curl -s https://smartdrip-system.agrinex.io/api/v1/devices/1/battery-history?period=week | jq
   curl -s https://smartdrip-system.agrinex.io/api/v1/devices/1/sleep-history?period=week | jq
   curl -s https://smartdrip-system.agrinex.io/api/v1/devices/1/irrigation-sessions?period=today | jq

4. Monitor logs:
   tail -f storage/logs/laravel.log | grep "DeviceDetail"

5. Verify UI di browser:
   https://smartdrip-system.agrinex.io/node/1
   - Buka Developer Console (F12)
   - Harusnya tidak ada error 404/500
   - Chart akan tampil atau show empty state

⚠️  Cache Layer dinonaktifkan sementara
    Setelah API stable dan verified di production, aktifkan kembali
    cache dengan uncomment CacheService wrapper di controller.

⚠️  Database Connection
    Jika masih ada timeout issue, check .env database credentials
    dan pastikan remote database server accessible.

⚠️  Irrigation Tables
    Jika irrigation_logs dan valve_logs belum ada, sistem akan
    gracefully return empty data. Table akan auto-created saat
    device mulai melakukan irigasi pertama kali.

✅  Empty Data adalah Normal
    Sistem IoT baru atau device baru online mungkin belum punya
    historical data. Ini bukan error condition.
